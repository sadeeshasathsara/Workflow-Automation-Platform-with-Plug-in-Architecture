# Security Policy

We take security seriously and appreciate constructive contributions to keeping the Workflow Automation Platform secure.

---

## Supported Versions

Only the latest release version on the `main` branch is actively supported with security updates.

---

## Reporting a Vulnerability

**Please do not report security vulnerabilities through public GitHub issues.**

If you discover a security vulnerability in this project, please report it by opening a private draft security advisory under the **Security** tab of this repository, or by contacting the repository administrators directly.

### What to Include
When reporting a vulnerability, please provide the following details:
- A clear description of the vulnerability and its potential impact.
- Step-by-step instructions or a Proof of Concept (PoC) script to reproduce the issue.
- Details of the environment (e.g., OS version, Python version, dependencies) in which it was tested.

### Our Commitment
Once a vulnerability report is received:
- We will acknowledge receipt of the report within 48 hours.
- We will work to investigate and verify the vulnerability.
- A fix will be prepared and merged into the repository.
- We will coordinate a release and publicly disclose the issue with proper credit to the reporter.
